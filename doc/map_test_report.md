## 🐛 核心 Bug：NodeFactory 首尾行强制硬编码为 FIGHT

**严重程度：高**
**位置：** `NodeFactory.java:46-60`

### 问题描述
`NodeFactory.generateNodes()` 中，第一行（row=0）和最后一行（row=endRow）的非起点/非终点节点全部强制设为 `NodeType.FIGHT`，而不是按权重随机生成。

```java
// 第一行
if (row == 0) {
    if (col == config.getStartCol()) {
        node = new Node(config.getStartNodeType(), row, col);
    } else {
        node = new Node(NodeType.FIGHT, row, col);  // ❌ 强制战斗
    }
}
// 最后一行
else if (row == config.getEndRow() && config.getEndNodeType() != null) {
    if (col == width / 2) {
        node = new Node(config.getEndNodeType(), row, col);
    } else {
        node = new Node(NodeType.FIGHT, row, col);  // ❌ 强制战斗
    }
}
```

### 影响
- 每层地图的第一行和最后一行几乎全是战斗节点
- 地图多样性大幅降低：5层共 27+25+24+28+28 = 132 个格点，约 3/5 被强制战斗
- 玩家在起点行和终点行完全无法遇到事件/商店/奖励/安全屋
- 与 `game_encyclopedia.md` 描述的随机地图理念不符

### 修复建议
第一行和最后行也应使用 `randomNodeType(config.getNodeWeights())` 生成，只对起点/终点格点做特殊类型指定。

## 🐛 Bug B：命运抉择(DECISION)被当作随机事件处理

**严重程度：中**
**位置：** `BackendDebugLauncher.java:137-151` (`resolveNode` 方法)

### 问题描述
`resolveNode()` 中，DECISION 类型的节点与其他非战斗/非商店/非奖励/非安全屋节点一样，落入 `else` 分支调用 `resolveEvent()`，被当作随机事件处理。

```java
} else {
    resolveEvent();  // DECISION 被当作随机事件！
}
```

而从 `game_encyclopedia.md` 得知，命运抉择应是**固定事件**：
> **第一层命运抉择 — 迷雾：**
> - **前进**：突破迷雾，进入森林
> - **回头**：结束游戏（获得隐藏结局「讲述中断」）

### 影响
- 玩家在终点决策节点无法获得前进/回头的选择
- 隐藏结局「讲述中断」无法通过调试器触发
- 第一层终点变成随机事件（如本次测试触发了"人生意义"）

### 修复建议
为 DECISION 节点添加专用的处理方法，展示"前进/回头"选项。

## 🐛 Bug C：地图终点设计不完整

**严重程度：低**
**位置：** `MapConfig.java`

### 问题描述
根据 `game_encyclopedia.md` 附录的游戏流程：

```
荒野 → 命运抉择(前进/回头) → 森林 → 诡异秘林(Boss) → 迷雾(Boss) → 命运抉择 → 高塔(Boss) → 结局
```

但当前 MapConfig 中，只有第1层（荒野）的终点是 DECISION，第4层（迷雾）的终点是 BOSS，缺少流程图中迷雾之后的第二个命运抉择。

### 影响
- 迷雾后直接进入高塔，缺少"命运抉择"的过渡
- 隐藏结局触发条件不完整
