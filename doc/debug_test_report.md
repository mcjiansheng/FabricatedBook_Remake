## 发现的 Bug

### 🐛 Bug #1: `deck` 命令始终显示 0/0/0

**严重程度:** 中

**现象:**
在地图命令提示符下输入 `deck`，始终输出：
```
牌堆: 抽牌堆 0 / 手牌 0 / 弃牌堆 0
```

**复现步骤:**
1. 启动 `BackendDebugLauncher`
2. 在 `>` 提示符后输入 `deck`

**原因分析:**
`BackendDebugLauncher.run()` 中调用了 `CardPool.getCardsByProfession("warrior")` 初始化卡池，但返回值从未赋值到 `player` 对象。`Player` 的 `drawPile`、`hand`、`discardPile` 列表初始化后为空。战斗系统通过 `CombatEngine.initBattle()` 内部初始化手牌，但地图层无法查看。

**影响:** 玩家在地图阶段无法了解当前牌组构成和牌堆数量。

**修复建议:** 在创建 Player 后，调用类似 `player.setDeck(CardPool.getCardsByProfession("warrior"))` 的方法初始化玩家牌组。

---

### 🐛 Bug #2: 敌人意图(Intent)与实际行动不匹配

**严重程度:** 低

**现象:**
Boss 战斗中，敌人显示意图 `ATTACK`，但实际执行的是格挡动作。例如：

```
敌人:
  1. 命令行首领 HP 70/70 格挡 0 意图 ATTACK
  ...
[CombatEngine] 命令行首领 行动: def8
```

**复现步骤:**
1. 进入任何 Boss 战斗
2. 观察每回合初显示的敌人意图
3. 对比回合末敌人实际执行的动作

**原因分析:**
`Enemy.getIntent()` 可能返回一个固定标签（如 `"ATTACK"`），而敌人的实际动作由动作脚本列表 `List.of("atk8", "def6", "atk10")` 决定。两套系统未同步。意图标签显示与脚本驱动力分离。

**影响:** 玩家无法根据意图做出策略决策，降低了战斗策略性。

**修复建议:**
- 在 `CombatEngine` 每回合开始时，根据敌人的当前脚本动作设置意图
- 或至少让 `getIntent()` 返回一个与脚本动作一致的描述

---

### 🐛 Bug #3: 战斗胜利回复值不一致

**严重程度:** 低

**现象:**
不同战斗的胜利回复值差异较大，未观察到与战斗难度或回合数的明显关联：

| 战斗类型 | 金币奖励 | HP回复 |
|:--|--:|--:|
| 普通战斗（第1层） | 27 | 1 |
| 精英战斗（第1层） | 32 | 8 |
| 普通战斗（第2层） | 22 | 8 |
| 普通战斗（第3层） | 25 | 1 |
| Boss（第3层） | 39 | 6 |
| Boss（第4层） | 39 | 7 |
| 普通战斗（第5层） | 21 | 11 |
| Boss（第5层-最终） | 35 | 10 |

**影响:** 奖励数值缺乏可预期的平衡逻辑。

**修复建议:** 为 `CombatEngine.getBattleReward()` 增加基于难度等级和敌人类型的计算公式。

---

### 🐛 Bug #4: `play` 命令后手牌索引混乱

**严重程度:** 低

**现象:**
使用 `play <编号>` 出牌后，手牌重新编号（移除已出的牌），导致玩家难以跟踪后续出牌的目标索引。

**复现步骤:**
1. 战斗中输入 `play 3` 打出第3张手牌
2. 原第4张牌变为新的第3张，而玩家若继续输入 `play 4` 可能打错牌

**影响:** 降低了手动操作的精确性，尤其在多张同类型牌时容易出错。

**修复建议:** 在每张手牌前增加一个稳定的内部 ID 显示，或在出牌后不重新编号（保留原编号但标记已出）。
