# 前端设计文档

本文档基于当前代码完整梳理前端界面结构、视觉规格和交互逻辑。坐标、尺寸均以游戏设计分辨率 `1280 x 720` 为准；桌面启动器固定窗口尺寸，不允许用户拖拽缩放。渲染框架为 LibGDX Scene2D + ShapeRenderer/SpriteBatch，所有屏幕使用 `FitViewport(1280, 720)`。

## 1. 全局规格

### 1.1 运行窗口

- 窗口标题：`Fabricated Book`
- 窗口尺寸：`1280 x 720`
- 最小窗口尺寸：`960 x 540`
- 推荐最大窗口尺寸：`3840 x 2160`
- 帧率：前台 60 FPS
- VSync：开启
- 是否可缩放：是
- 设计坐标系：固定为 `1280 x 720`，窗口缩放、全屏和无边框模式下通过统一 viewport 等比缩放 UI。
- 显示模式快捷键：
  - `F11` 或 `Alt + Enter`：在窗口模式和全屏模式之间切换。
  - `F10`：在普通窗口和无边框窗口之间切换。

### 1.2 坐标体系

- 文档中未特别说明时，坐标为 LibGDX 世界坐标：左下角为 `(0, 0)`，右上角为 `(1280, 720)`。
- 鼠标输入来自屏幕坐标时，代码会将 y 轴反转为世界坐标。
- `Stage` 负责 UI 控件与点击事件；部分背景、地图连线、血条等由 `ShapeRenderer` 或 `SpriteBatch` 手动绘制。

### 1.3 字体

- 全局字体：`core/src/main/resources/ys_zt.ttf`
- 逻辑基础字号：18
- 字体贴图按逻辑字号缓存生成：`game.getFont()` 返回 `18px` 正文字体，`game.getFontForScale(scale)` 返回 `round(18 * scale)` 的独立字体贴图。
- 字体纹理过滤：`Linear`；不使用 mipmap，避免大字边缘被多级采样柔化。
- 实现规则：大于 `1.0` 的标题、回合数、奖励标题等应使用对应字号字体，不直接放大 `18px` 正文字体；小于 `1.0` 的紧凑文本可以缩小正文字体。
- 字符集来源：
  - 基础中文、英文、数字、常用标点。
  - 卡牌、藏品、药水、事件、地图、怪物 JSON 数据中的动态文本。
- 字体对象按字号共享缓存；自定义 Actor 若临时设置 `font.getData().setScale(...)`，必须在绘制后恢复缩放，避免污染后续绘制。

### 1.6 公共 UI 主题与资源生命周期

- `view/ui/UiTheme` 是新增的全局视觉 token 入口，当前统一管理战斗/地图背景、生命与金色强调、以及按钮的默认、悬停、按下和边框颜色。后续页面不得继续为已有 token 复制新的颜色常量。
- `view/ui/UiStyles` 缓存按钮 NinePatch 的运行时纹理。它由 `FabricBookGame.dispose()` 在应用退出时统一释放；Screen 不应自行释放这些全局样式资源。
- 当前阶段只完成颜色与按钮资源收口。面板边距、字体层级、Tooltip、模态与滚动容器将随商店、事件和战斗界面的迁移继续接入公共组件。
- `UiModal` 统一提供全屏阻断遮罩、居中面板与关闭生命周期；商店弃牌确认框和战斗失败弹窗已使用该容器，战斗奖励等特殊展示后续按同一结构迁移。
- `UiFeedback` 提供信息、成功、警告、错误四种短暂反馈语义；商店购买、弃牌与药水栏更新已使用它，避免不同操作只修改无色静态提示文字。
- `UiLayout` 集中定义 HUD 高度、HUD 后页面起始位置、页面边距、面板内边距、常用按钮尺寸与分区间距；安全屋、总览、商店模态和 HUD 已迁移到这些 token。

### 1.7 商店信息结构

- 商店内容使用可纵向滚动的商品面板，按“卡牌 / 藏品 / 药水 / 服务”四个区块显示，而非将不同商品混入同一文本列表。
- 每个商品行必须同时显示名称、价格、效果说明和购买状态；已购商品显示“已售罄”。
- 顶部持续显示当前金币与即时反馈。金币不足、药水栏已满、购买成功与一般购买失败均应提供可读提示；不可购买的商品按钮直接显示“金币不足 / 药水栏满 / 已售罄”，而非保留可点击的“购买”。
- “移除一张卡牌”服务会打开牌组选择模态，展示价格、可移除卡牌、取消与确认操作；确认后同步更新金币、牌组和商店反馈。

### 1.8 事件信息结构

- 事件页分为叙事面板与选项/结果面板，两者在 1280×720 设计画布内留出明确间距；文本与按钮宽度必须扣除面板内边距，不能溢出到相邻面板。
- 事件标题使用金色强调，叙事与选项正文使用高对比白色；选择后以独立结果状态替换选项，再提供“继续前进”。
- 结果状态在叙事文本下方单列实际变动摘要（金币、生命、获得藏品），并即时刷新顶部 HUD，避免玩家只从长描述中猜测结算。
- 事件规则、代价和结果仍由 `EventHandler` 负责；界面只负责呈现和触发，不在 Screen 内新增事件数值逻辑。

### 1.9 地图总览入口

- 地图顶栏的“卡牌”和“藏品”使用独立 Scene2D 按钮，不再依赖地图手工坐标命中测试。
- 两个入口分别打开卡牌或藏品总览；总览使用三列可滚动网格与固定详情栏，并保留返回地图路径。
- 卡牌格显示费用、类型与稀有度；藏品格显示名称与稀有度。点击格子只更新详情栏，不改变牌组或藏品状态。

### 1.9.1 对局统一状态栏

- `GameHud` 是地图、事件、商店、战斗、安全屋与库存页唯一的对局状态栏：左侧显示职业、生命和金币；中间显示药水栏；右侧依次显示层数、卡牌与藏品入口。
- 药水操作浮层锚定于被点击的药水按钮下方，不使用全屏遮罩；战斗可使用或丢弃，非战斗只能丢弃。
- 地图的楼层开屏动画是唯一不显示该 HUD 的对局过场，动画结束后恢复统一状态栏。
- 战斗的回合标题与即时结算反馈位于 HUD 下方的独立信息区，不能覆盖药水栏或顶部入口。

### 1.10 奖励与药水规则

- 奖励可逐项领取，也可在任意时刻通过“离开奖励”继续流程；卡牌奖励始终提供“跳过”。
- 药水奖励在药水栏满时不应静默消失：玩家可选择丢弃一瓶现有药水后领取，或跳过该奖励。
- 顶部药水栏直接显示每瓶药水名称。点击一瓶药水后，在该按钮正下方打开不遮罩主画面的紧凑操作浮层；战斗中显示“使用 / 丢弃 / 取消”，非战斗中只显示“丢弃 / 取消”。
- `PotionActionBar` 复用于地图、商店、事件和战斗的药水管理入口；丢弃后立即自动保存对局状态。

## 2. 通用控件

### 2.1 文本按钮

普通 Scene2D `TextButton` 使用 `UiStyles.buttonStyle(...)`，由灰色底框、黑色描边和文字组成，不直接裸露文字。按钮状态：

- 默认背景：`(0.36, 0.36, 0.36, 1)`
- 悬停背景：`(0.46, 0.46, 0.46, 1)`
- 按下背景：`(0.24, 0.24, 0.24, 1)`
- 文本颜色：白色；按下时为浅灰。

常见按钮规格：

- 主流程按钮：`220 x 52` 或 `220 x 56`
- 小按钮：`150 x 42`、`150 x 48`、`180 x 50`
- 战斗结束回合按钮：`130 x 50`
- 商店购买按钮：`80 x 30`

### 2.2 自定义按钮 `ButtonActor`

当前项目中保留了一个独立的自定义按钮 Actor，默认尺寸 `200 x 50`。

- 普通背景：`(0.3, 0.35, 0.4, 1)`
- 悬停背景：`(0.4, 0.5, 0.6, 1)`
- 按下背景：`(0.2, 0.25, 0.3, 1)`
- 文字：白色，居中
- 交互：
  - 鼠标进入：进入悬停状态。
  - 鼠标离开：恢复普通状态。
  - 按下：进入按下状态。
  - 抬起：触发回调并恢复悬停/普通状态。

### 2.3 卡牌 Actor

卡牌尺寸固定为 `120 x 170`。卡牌不会随文字内容改变大小。

#### 视觉结构

- 底色：`(0.95, 0.93, 0.85, 1)`
- 类型图标：`20 x 20`，位置为卡牌左上角内缩约 `8 px`
- 名称文字：
  - x：卡牌左侧 `32 px`
  - y：卡牌顶部下方约 `10 px`
  - 颜色：深褐黑 `(0.12, 0.10, 0.08, 1)`
  - 字体缩放：约 `0.78`
- 类型/稀有度：名称下方显示中文类型与稀有度，例如“攻击 · 普通”。
- 消耗文字：`用 N`；X 费牌显示为 `用 X`
  - x：卡牌左侧 `8 px`
  - y：顶部下方约 `42 px`
- 描述文字：
  - 按实际字体宽度换行，兼容中文、英文、数字和混合标点；不再以固定字符数切分
  - 最多 4 行，为类型/稀有度和底部状态预留空间
  - 起始 y：顶部下方约 `72 px`
  - 行距：约 `18 px`
  - 超出时最后一行以 `...` 结尾
- 底部状态文字：
  - 当 `value > 0` 时显示 `值 N`
  - 无法打出或能量不足时显示原因，并使用半透明暗层弱化卡面；该状态不可拖拽。
  - x：卡牌左侧 `8 px`
  - y：底部上方约 `14 px`

#### 边框颜色

- 攻击牌：红色 `Color.RED`
- 防御牌：蓝紫 `(0.42, 0.42, 1, 1)`
- 技能牌：绿色 `(0.35, 0.95, 0.40, 1)`
- 装备牌：金色 `Color.GOLD`

#### 悬停与拖拽

- 鼠标悬停：卡牌缩放到 `1.08`
- 拖拽时：缩放到 `1.06` 并置顶
- 拖拽时覆盖一层半透明金色：`(1, 0.8, 0.2, 0.5)`
- 右键点击正在拖拽的卡牌：取消拖拽并回到原位

  - 保持原图片长宽比，不拉伸变形。
  - 绘制中心对齐节点坐标。
- 透明度：
  - 已走过节点：正常显示，整体略暗
  - 当前可选节点：完全不透明
  - 未来节点：透明度约 `0.82`

边框规则：

- 当前可选择节点：金色边框，颜色 `(1, 0.82, 0, 1)`，围绕图片外扩 `6 px`，边框厚度 `4 px`。
- 已走过或当前所在节点：黑色边框，围绕图片外扩 `6 px`，边框厚度 `4 px`。
- 未来未到达节点：无边框。

### 5.5 连线

- 连线在节点下方先绘制，但端点停在节点图片边缘，不从图片主体下方穿过。
- 连线宽度：`3.5 px`，使用填充矩形线段绘制，避免高分辨率下变成过细的 1px OpenGL 线。
- 可通向当前可选节点或从当前可选节点延伸出的线：黑色，透明度约 `0.86`
- 其他未来线路：黑色，透明度约 `0.62`

### 5.6 节点类型

- 战斗：普通战斗，胜利获得金币、卡牌奖励，并可能额外获得藏品/药水。
- 紧急作战：使用紧急作战怪物组，奖励逻辑同战斗结算，怪物池优先选择 `emergency` 组。
- Boss：选择 Boss 怪物组。
- 不期而遇：随机事件。
- 得偿所愿：固定进入 `好诗歪诗` 事件。
- 诡异行商：进入商店。
- 命运抉择：固定进入 `投资` 事件。
- 安全屋：固定进入 `人生意义` 事件。

### 6.9 战斗流程

进入战斗时：

1. 初始化战斗引擎。
2. 如果玩家抽牌堆为空，则生成初始战士基础牌组：5 张攻击、5 张防御。
3. 洗牌。
4. 触发藏品战斗开始效果。
5. 自动开始第 1 回合。

每回合开始：

1. 回合数 +1。
2. 能量重置为最大能量，并叠加额外能量状态。
3. 没有装甲状态时清空格挡。
4. 触发玩家和敌人的回合开始 Buff。
5. 清理过期 Buff。
6. 抽 5 张牌；如果玩家眩晕，则不抽牌并清除眩晕。
7. 更新敌人意图。

使用卡牌：

1. 检查能量；X 费牌使用当前全部剩余能量作为实际消耗。
2. 从手牌移除对应实例。
3. 扣除能量。
4. 解析卡牌效果并入队。
5. 非消耗牌进入弃牌堆；消耗牌进入消耗牌堆。
6. 执行动作队列。
7. 触发藏品卡牌使用事件。
8. 检查胜负。

结束回合：

1. 处理玩家剩余手牌：虚无牌进入消耗牌堆，保留牌留在手牌，其余牌进入弃牌堆。
2. 敌人按当前意图行动。
3. 触发回合末效果。
4. 若战斗未结束，开始下一回合。

  - 已购买时追加 `[已购]`
  - 未购买颜色：白色
  - 已购买颜色：灰色
  - 单元格内边距：`5`
- 商品描述：

  - 自动换行
  - 宽度：约 `420`

  - 尺寸：`80 x 30`
  - 左边距：`10`
  - 已购买商品不显示购买按钮。

### 8.4 商品生成规则

商店进入时生成：

- 7 张职业卡牌。
- 3 个非诅咒、玩家未拥有的藏品。
- 3 个药水。
- 1 次弃牌机会。

价格规则：

- 价值为 1 的卡牌：`50-70` 金币。
- 其他卡牌：`90-110` 金币。
- 藏品：`稀有度数值 * 25 + 55-70` 金币。
- 药水：`30-70` 金币。
- 弃牌：基础 `75` 金币，每次购买后全局价格 +25。

### 8.5 购买逻辑

点击商品的 `购买`：

- 金币不足：无效果。
- 已购买：无效果。
- 卡牌商品：扣金币，将卡牌复制后加入玩家抽牌堆。
- 藏品商品：扣金币，加入藏品管理器。
- 药水商品：
  - 药水栏未满：扣金币，加入玩家药水栏。
  - 药水栏已满：购买失败并退还金币。
- 成功购买后：
  - 商品标记为已购。
  - 刷新金币文本。
  - 重新渲染商品列表。

### 8.6 离开商店

- 按钮：
  - 文本：`离开商店`
  - 位置：`x=1080, y=30`
  - 尺寸：`180 x 50`
- 点击后返回地图并完成当前节点。

### 8.7 当前已知限制

`ShopManager` 已实现弃牌购买逻辑 `purchaseRemove(cardIndex)`，但当前 `ShopScreen` 只显示弃牌机会文字，没有提供选择卡牌并确认移除的完整 UI。

## 10. 资源释放

各屏幕在 `dispose()` 中释放自己持有的 `Stage` 和贴图资源。

- `TitleScreen`、`CharacterSelectScreen` 释放背景与角色贴图。
- `MapScreen` 释放节点贴图。
- `BattleScreen` 释放舞台、ShapeRenderer、实体贴图由 Actor 自行释放。
- `EventScreen`、`ShopScreen`、`InventoryScreen` 释放 Stage 和 ShapeRenderer。

注意：全局 `SpriteBatch` 和全局字体由 `FabricBookGame.dispose()` 统一释放。

## 11. 全流程交互路径

### 11.1 新游戏

1. 标题界面点击 `开始游戏`。
2. 角色选择界面点击一个职业。
3. 进入第 1 层地图。
4. 点击金框标记的可达节点。
5. 根据节点进入战斗、事件或商店。
6. 完成节点后返回地图。
7. 完成本层终点后进入下一层。

### 11.2 战斗胜利

1. 玩家通过拖拽卡牌、使用药水、结束回合进行战斗。
2. 所有敌人死亡后触发胜利。
3. 显示胜利奖励弹窗。
4. 点击 `继续`。
5. 进入卡牌奖励选择。
6. 点击卡牌获得奖励，或点击 `跳过`。
7. 返回地图并完成节点。

### 11.3 战斗失败

1. 玩家生命降到 0。
2. 显示战斗失败弹窗。
3. 点击 `重新开始`。
4. 返回标题界面。

### 11.4 事件节点

1. 地图点击事件类节点。
2. 事件界面左侧显示标题和描述。
3. 右侧显示可选项。
4. 点击一个选项后立即执行结果。
5. 右侧切换为结果文本和 `继续前进`。
6. 点击继续后返回地图。

### 11.5 商店节点

1. 地图点击商店节点。
2. 商店界面生成商品。
3. 点击 `购买` 尝试购买商品。
4. 成功购买后商品变为 `[已购]`。
5. 点击 `离开商店` 返回地图。

### 11.6 查看卡牌/藏品

1. 地图顶部点击 `卡牌` 或 `藏品`。
2. 进入总览界面。
3. 左侧查看卡牌，右侧查看藏品。
4. 点击 `返回地图` 回到地图。

## 12. 当前前端待完善点

这些项目不是文档缺失，而是当前代码实现中的真实限制，后续 UI 迭代应优先处理：

- 标题界面的 `继续游戏` 读取失败时没有用户可见提示。
- 地图最后一层完成后没有独立通关结算界面。
- 商店弃牌功能后端已存在，但前端没有完整选择卡牌并移除的界面。
- 卡牌奖励界面只有卡牌文字卡，没有更复杂的卡牌稀有度/品质装饰。
- 卡牌总览当前是文本列表，不是卡牌实体网格。
- 藏品总览当前是文本列表，没有图标格子。
- 事件选项中的 emoji 会被 `cleanLabel` 清理，当前界面不显示图标化选项。
- 战斗状态提示是单行文本，长文本可能需要进一步裁剪或分行策略。
