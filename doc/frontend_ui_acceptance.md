# 前端 UI 验收矩阵

本清单用于验证 P1 前端架构与信息结构。每次影响 UI 的提交至少执行 `./gradlew test`，并对受影响行使用对应 CLI 入口和 Computer Use 截图复核。

## 基准

- 设计画布：`1280 × 720`。
- 最小窗口：`960 × 540`；不得出现不可点击的关键入口或 HUD 覆盖主按钮。
- 所有对局界面（地图、战斗、商店、事件、安全屋、总览）使用相同 `GameHud`：左侧职业/生命/金币，中部药水，右侧楼层、卡牌、藏品。
- 地图楼层开屏动画例外：动画过程中不显示 HUD。

## 状态矩阵

| 场景 | CLI 参数 | 必查项目 | 通过标准 |
|---|---|---|---|
| 标题 | `title` | 长标题、继续游戏、按钮层级 | 主按钮可见且文字不溢出；继续失败有可读反馈 |
| 地图 | `map` / `map:3` | HUD、可达节点、已完成节点、拖动 | 金框表示可前进；已完成节点与未访问节点可区分；HUD 不遮挡地图 |
| 楼层开屏 | `map` 后立即截图 | 过场遮罩 | HUD 与库存入口隐藏；层名居中且退出后恢复 HUD |
| 战斗 | `battle` | 手牌、能量、敌人意图、拖拽目标 | 卡面含费用/类型/稀有度；能量不足和不可打出态明确；拖向目标时有高亮/预览 |
| 战斗失败 | `battle-defeat` | 模态阻断、返回路径 | 深色遮罩阻断战场；失败面板居中；重新开始可用 |
| 奖励满药水 | `reward-potion` | 部分领取、跳过、丢弃后领取 | 可离开奖励；卡牌可跳过；满栏时能丢弃现有药水或跳过 |
| 商店 | `shop` | 商品分区、购买状态、长列表、弃牌模态 | 金币不足/药水栏满/售罄不可点击且说明原因；购买或弃牌有彩色反馈 |
| 事件 | `event` / `--screen=event --event=投资` | 叙事/选项/结果过渡 | 选项不溢出；选择后显示结果摘要和继续入口；HUD 同步数值 |
| 总览 | `inventory` | 三列网格、详情栏、长牌组 | 网格可滚动；点击条目只更新详情；返回路径稳定 |

## 边界数据

- 长中文、英文与数字混合的卡牌和商品说明：不裁掉关键规则；卡面允许省略时必须有详情入口或预览。
- 药水栏满、卡牌手牌偏多、多敌人、长商品列表：重要操作不相互遮挡。
- 模态叠加：Esc/取消只能关闭当前模态；背景不能继续接收玩法输入。
- 鼠标/键盘：Esc 为统一返回或关闭菜单；右键取消拖拽；滚轮只滚动有滚动容器的内容。
