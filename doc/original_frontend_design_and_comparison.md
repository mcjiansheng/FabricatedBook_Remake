# 原项目前端设计文档与重制版对比

本文档基于原项目 `mcjiansheng/FabricatedBook` 的源码阅读结果编写，分析对象为临时克隆仓库 `/private/tmp/FabricatedBook_original`，提交版本 `cf06f4b`。原项目使用 C + SDL2/SDL_image/SDL_ttf 实现 UI，主入口为 `game.c`，界面逻辑分布在 `tools/button.c`、`tools/gamecreater.c`、`tools/map.c`、`tools/fight.c`、`tools/events.c`、`tools/shop.c`、`tools/card.c`。

坐标除特别说明外均以原项目默认窗口 `1920 x 1080` 为基准；SDL 坐标原点在左上角，x 向右增加，y 向下增加。

## 1. 原项目全局 UI 规格

### 1.1 技术栈

- 窗口与渲染：SDL2
- 图片加载：SDL_image
- 字体渲染：SDL_ttf
- 字体文件：`./res/ys_zt.ttf`
- 图像资源目录：`./img/`
- 默认窗口标题：`The Explorer and Fabricated Book`
- 默认窗口尺寸：`1920 x 1080`
- 窗口创建参数：`SDL_WINDOW_SHOWN`
- 渲染器：`SDL_RENDERER_ACCELERATED`
- 原项目支持窗口尺寸变化，部分按钮会按窗口比例重新计算，但大量实体、卡牌、地图点位仍使用固定像素坐标，因此实际设计基准应视为 `1920 x 1080`。

### 1.2 坐标与布局方式

原项目混合使用两套布局方法：

- 比例按钮：`Button.rec` 使用窗口比例 `{x, y, w, h}`，通过 `updateButton` 转换为像素矩形。
- 固定绘制：地图节点、战斗实体、手牌、奖励项、商店商品、事件选项大量使用固定像素坐标。

按钮字体大小会随窗口高度变化：

```text
font_size_actual = (window_h - 600) / 20 + base_font_size
```

### 1.3 颜色常量

原项目颜色定义在 `tools/Color.h`：

- 白色：`rgba(255,255,255,255)`
- 浅红：`rgba(255,128,128,255)`
- 浅蓝：`rgba(128,128,255,255)`
- 浅绿：`rgba(128,255,128,255)`
- 浅灰：`rgba(240,240,240,255)`
- 背景灰：`rgba(170,170,170,170)`
- 灰色按钮：`rgba(100,100,100,100)`
- 黑色：`rgba(0,0,0,0)`
- 灰绿：`rgba(72,112,74,100)`
- 灰红：`rgba(178,73,74,100)`
- 深灰：`rgba(50,50,50,50)`
- 红色：`rgba(255,0,0,255)`
- 金色：`rgba(255,215,0,255)`
- 暗黄：`rgba(204,204,0,255)`
- 橙色：`rgba(255,165,0,0)`
- 紫色：`rgba(128,0,128,255)`

注意：部分颜色 alpha 不是 255，但 SDL 默认未统一启用混合模式，因此实际显示可能接近不透明色块。

### 1.4 通用按钮

按钮结构包含：

- 相对矩形：`rec.x, rec.y, rec.w, rec.h`
- 实际矩形：`rect.x, rect.y, rect.w, rect.h`
- 普通色、悬停色、按下色
- 文本与字体

绘制规则：

- 按下时使用 `pressColor`
- 悬停时使用 `hoverColor`
- 普通时使用 `color`
- 文本使用白色，水平垂直居中

鼠标逻辑：

- `SDL_MOUSEMOTION` 更新悬停状态。
- `SDL_MOUSEBUTTONDOWN` 设置按下状态。
- `SDL_MOUSEBUTTONUP` 若仍在按钮范围内，则触发对应行为。

- 每完成当前层尾节点并进入下一层时。

### 5.2 视觉结构

- 背景：灰色 `rgba(220,220,220,220)`

### 6.4 地图生成

共有 6 个 Layer：

- 第 1 层：宽 3、长 4。
- 第 2 层：宽 3、长 5。
- 第 3 层：宽 4、长 6。
- 第 4 层：宽 4、长 7。
- 第 5 层：宽 4、长 7。
- 第 6 层：固定线路，长 4，节点依次为商店、紧急作战、安全屋、Boss。

前五层每列节点数随机：

- 起点列固定 1 个。
- 终点列固定 1 个。
- 中间列为 `1..width` 个节点。

节点生成概率：

- 第 1 层：战斗 60%，紧急作战 10%，不期而遇 30%。
- 第 2-5 层：战斗 40%，紧急作战 20%，不期而遇 10%，奖励 10%，商店 10%，安全屋 10%。
- 第 6 层：固定。

特殊起终点：

- 第 1 层：起点战斗，终点命运抉择。
- 第 2 层：起点战斗，终点商店。
- 第 3 层：起点奖励，终点 Boss。
- 第 4 层：起点紧急作战，倒数第二列 Boss，最后一列命运抉择。
- 第 5 层：起点不期而遇，终点 Boss。

### 6.5 地图节点位置

节点中心由 `init_map_printer` 计算：

- 起始 x：`200`
- 横向步长：`500`
- y 初始为 `windowHeight / 2`
- 若该列 3 个节点：
  - y 步长：`windowHeight / 4`
  - 第一节点 y：`windowHeight/2 - step_y`
- 若该列 2 个节点：
  - y 步长：`windowHeight / 3`
  - 第一节点 y：`windowHeight/2 - step_y/2`
- 若该列 4 个节点：
  - y 步长：`windowHeight / 5`
  - 第一节点 y：`windowHeight/2 - step_y*1.5`
- 若该列 1 个节点：
  - y：`windowHeight / 2`

节点图标绘制：

- 根据节点类型从 `node_texture[1..9]` 加载。
- 使用贴图原始宽高。
- 将贴图中心对齐到节点中心。

节点点击/高亮矩形：

- 以节点贴图大小为基础，宽高各增加 `20 px`。
- 矩形中心对齐节点中心。

### 6.6 节点高亮

`print_nodes` 在节点图标之前绘制高亮底板：

- 当前节点：黑色矩形。
- 可选下一节点：金色矩形。
- 初始状态没有当前节点时，起点使用金色矩形。

高亮矩形比节点贴图大 `20 px`，作为外框/背景标记。

### 6.7 地图拖拽

- 鼠标按下后设置 `dragging = true`。
- 鼠标移动时：
  - `delta_x = event.motion.x - mouse_x`
  - 地图水平偏移 `x += delta_x`
  - 更新鼠标坐标。
- 地图只支持横向拖拽，没有纵向滚动。
- 节点点击使用当前偏移 `x` 进行命中检测。

### 6.8 节点进入逻辑

- 如果当前节点为空，只允许点击本层起点。
- 如果当前节点不为空，只允许点击当前节点的下一节点。
- 进入节点后：
  - `last_node = node`
  - 总节点计数 +1
  - 根据层级规则修改玩家状态：
    - 第 2 层：进入非战斗节点损失 10-20 金币，最低为 0。
    - 第 3 层：进入非战斗节点 `extra_damage +1`，进入战斗节点 `extra_damage -1`，范围 -3 到 3。
    - 第 4 层：每步随机回复 5-30 或损失 1-20 生命，最低保留 1。
    - 若拥有特定藏品且进入非战斗节点，获得 20 金币。
  - 根据节点类型切换到战斗、事件、奖励、商店或安全屋。

### 6.9 药水交互

地图、事件、商店、奖励界面都复用药水栏选择逻辑：

- 点击药水矩形：选中药水。
- 显示描述框：
  - 位置：选中药水 x 坐标，`y=150`
  - 尺寸：`300 x 100`
  - 背景：白色
  - 描述文字：黑色
- 地图/事件/商店/奖励界面通常只绘制 `丢弃` 按钮，不绘制 `使用` 按钮。
- 点击 `丢弃`：移除该药水。
- 如果鼠标离药水区域足够远，会取消选择。

### 7.13 战斗流程

战斗开始：

1. 清空手牌、牌库、弃牌堆。
2. 初始化玩家和敌人的 Buff。
3. 从玩家总牌组复制到战斗牌库。
4. 洗牌。
5. 回合数归零。
6. 开始第一回合。

回合开始：

1. 回合数 +1。
2. 玩家 Buff 层数递减。
3. 抽最多 5 张牌。
4. 能量设为 3。
5. 法师在奇数回合额外 +1 能量。
6. 处理额外能量 Buff。
7. 没有装甲则清空格挡，有装甲则消耗一层装甲。
8. 应用若干藏品效果。

结束回合后：

1. 手牌全部进入弃牌堆。
2. 处理毒、凋零、不死等状态。
3. 按时间片执行敌人 1、2、3 行动。
4. 结算敌人 Buff 与玩家不死。
5. 约 700 帧计数后进入下一回合。

胜负检测：

- 玩家 hp <= 0：失败。
- 所有敌人 hp <= 0：胜利。

  - 特定藏品条件下：`endness = 2`

### 16.2 视觉结构

- 背景：灰色。
- 标题：
  - 字号：70
  - x：`windowWidth/2`
  - y：`200`
  - alpha 淡入
- 正文：
  - 字号：30
  - x：`windowWidth/2`
  - y：`300`
  - alpha 淡入
- 统计：
  - 字号：25
  - x：`windowWidth/2`
  - y：`windowHeight/2`
  - 内容：
    - 总节点数
    - 战斗次数
    - 获得卡牌数
    - 获得藏品数

结局内容：

- `讲述中断`：黑色标题。
- `故事终局`：浅蓝标题。
- `沉入虚妄`：紫色标题。

### 16.3 交互

- 淡入完成后等待用户按键抬起或鼠标抬起。
- 用户操作后结束结局界面。

## 17. 原项目与重制版前端差异

重制版参考文档：`doc/frontend_design.md`。

### 17.1 技术与分辨率

| 项目 | 原项目 | 重制版 |
|---|---|---|
| 技术栈 | C + SDL2 | Java + LibGDX Scene2D |
| 默认分辨率 | 1920x1080 | 1280x720 |
| 坐标原点 | 左上角 | 左下角 |
| 缩放策略 | 部分按钮比例缩放，主体固定像素 | FitViewport 固定设计尺寸 |
| UI 组织 | 每个界面手写 SDL 循环 | Screen/Stage/Actor 分层 |

### 17.2 视觉风格

- 原项目整体以浅灰背景为主，战斗、地图、事件、商店都是灰底黑字。
- 重制版战斗界面改为深色战斗场景，地图和事件保留浅灰风格。
- 原项目更接近开发原型：信息直接铺在画面上，按钮与实体没有统一面板系统。
- 重制版加入了顶部栏、手牌区、弹窗遮罩和实体血条，更接近现代卡牌 Roguelike UI。

### 17.3 主菜单与选角

- 原项目主菜单有背景图，只有开始和退出两个有效按钮；介绍和设置存在但未启用。
- 重制版主菜单有开始、继续、退出；支持存档继续。
- 原项目选角界面显示未解锁角色，并且只有选中后才出现开始按钮。
- 重制版选角目前三个职业均可选择，卡片布局更规整，并提供返回按钮。

### 17.4 地图

- 原项目地图使用 6 层，其中第 6 层是固定路线；重制版当前主要是 5 层结构。
- 原项目节点图标使用原始尺寸，节点间距很大，横向步长 500。
- 重制版节点统一为 `112 x 66` 底板和 `34 x 34` 图标，更密集。
- 原项目可选节点用金色实心底板标记，当前节点用黑色实心底板标记。
- 重制版可选节点用金色边框，已走过节点用黑色边框，未来节点不加框。
- 原项目地图只支持横向拖动；重制版支持横向和有限纵向拖动。
- 原项目有 `复原` 按钮用于地图偏移过大时归位；重制版没有该按钮，而是限制滚动范围。

### 17.5 战斗

- 原项目战斗是浅灰背景，实体和 UI 均固定坐标。
- 重制版战斗分成顶部栏、战斗区、手牌区，视觉层级更清楚。
- 原项目玩家立绘固定在 `{400,400,200,350}`，敌人固定在右侧三组位置。
- 重制版玩家和敌人使用 Actor 封装，按敌人数量动态布局。
- 原项目血量、格挡主要是文本；重制版加入血条和格挡条。
- 原项目状态图标为 `20 x 20`，起点在实体下方或顶部；重制版状态图标为 `26 x 26`，与血条和实体更紧密。
- 原项目卡牌交互是点击选牌再点击目标；重制版是拖拽卡牌到目标。
- 原项目结束回合按钮叫 `下一回合`；重制版叫 `结束回合`。
- 原项目敌人行动阶段有明显的阶段文字和时间片；重制版偏即时结算，状态提示较短。

### 17.6 卡牌显示

- 原项目战斗手牌尺寸 `150 x 225`，奖励/总览卡牌尺寸 `300 x 450`。
- 重制版所有卡牌 Actor 当前统一为 `120 x 170`。
- 原项目卡牌边框类型：
  - 攻击红
  - 防御浅蓝
  - 装备灰
  - 技能浅绿
- 重制版：
  - 攻击红
  - 防御蓝紫
  - 技能绿
  - 装备金
- 原项目卡牌文字使用 `费用: N`；重制版使用 `用 N`。
- 原项目卡牌奖励界面没有跳过；重制版提供 `跳过`。

### 17.7 药水

- 原项目药水栏在几乎所有主界面都存在，且地图、事件、商店、奖励界面都允许丢弃药水。
- 原项目战斗中可使用药水，非战斗界面大多只允许丢弃。
- 重制版地图顶部显示药水名称；战斗中提供药水按钮使用；事件/商店里目前没有统一药水弹窗交互。
- 原项目巫师药水上限为 5；重制版目前常规显示为 `0/3`。

### 17.8 奖励

- 原项目战斗奖励是“逐项点击领取”：
  - 金币、卡牌、药水、藏品均为可点击条目。
  - 点击卡牌条后进入卡牌选择。
  - 可提前点继续离开，未点奖励可能不会领取。
- 重制版胜利奖励是汇总弹窗 + 继续 + 卡牌选择，金币和额外奖励更偏自动发放。
- 原项目得偿所愿是事件式三选一藏品；重制版目前也映射为事件，但 UI 布局更靠近左右栏。

### 17.9 商店

- 原项目商店是卡牌、藏品、药水、弃牌以卡片形式直接铺在画布上，点击商品本身购买。
- 重制版商店是文本列表 + 购买按钮。
- 原项目弃牌 UI 完整：买弃牌后进入卡牌选择，选中后点确定删除。
- 重制版后端已有弃牌逻辑，但当前前端文档中标注为待完善。

### 17.10 事件

- 原项目事件有淡入动画，左侧事件文本，右侧选项黑/灰块；选中后进入结果淡入，结果完成后任意点击退出。
- 重制版事件界面也是左右结构，但结果阶段有明确 `继续前进` 按钮。
- 原项目事件界面仍能操作药水丢弃；重制版没有保留这一跨界面药水操作。

### 17.11 总览界面

- 原项目卡牌/藏品总览使用实际卡牌/藏品卡片网格展示，并支持滚轮和拖拽滚动。
- 重制版当前是两个 ScrollPane 文本列表，更清晰但缺少原项目的卡片视觉。

### 17.12 结局与失败

- 原项目有三类结局界面，并显示探索统计。
- 重制版当前战斗失败有独立弹窗，但完整通关/分支结局 UI 仍未实现。
- 原项目失败界面提供返回主菜单和退出游戏；重制版失败弹窗目前提供重新开始。

## 18. 对重制版后续 UI 还原建议

如果目标是更贴近原项目而不是完全现代化重构，建议优先补齐以下差异：

1. 地图层级：补上第 6 层固定路线和原项目中的分支 PPT/结局入口。
2. 商店：将商品从文本列表改为卡牌/藏品/药水卡片网格，并实现弃牌选择界面。
3. 总览：将卡牌与藏品总览从文本列表升级为卡片网格。
4. 奖励：确认是否保留原项目“逐项点击领取”的交互，还是沿用重制版自动领取。若要还原，应让金币、卡牌、药水、藏品都成为可点击奖励项。
5. 药水：明确非战斗界面是否允许丢弃药水，以及巫师 5 药水上限是否恢复。
6. 结局：补齐 `讲述中断`、`故事终局`、`沉入虚妄` 三类结局界面与统计。
7. 事件：保留淡入节奏、选项 hover/press 外框反馈，并决定结果阶段使用任意点击退出还是按钮退出。
8. 卡牌奖励：若严格还原原项目，应移除跳过按钮；若保留重制版便利性，则文档中标注为设计改动。
